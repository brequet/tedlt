## Quick Example

```sh
# Create a simple ticket
tedlt create "Update documentation"

# Create a bug with the 'bug' profile
tedlt create "Fix null pointer exception" --profile bug

# Create a critical bug (inherits from bug profile + adds urgency)
tedlt create "Production outage" --profile bug-critical

# Create a task for on another board (multiple profiles supported)
tedlt create "Design new feature" --profile proj2 --profile task

# Discover project information
tedlt info project MYPROJ

# View ticket details
tedlt info ticket MYPROJ-123
```

## How It Works

1. **Configure once**: Set up your Jira credentials and default settings
2. **Define profiles**: Create profiles for common ticket types (bugs, features, tasks)
3. **Create tickets**: Use profiles to create tickets with a single command
4. **Discover metadata**: Query Jira for project info, field IDs, and more

## What Makes tedlt Different?

- **Title-first philosophy**: Most tickets start with a good title. Fill in details later if needed.
- **Configuration as code**: Your ticket templates live in config files.
- **Composable profiles**: Combine multiple profiles for maximum flexibility.
- **Developer-friendly**: Built for command-line workflows and automation.
